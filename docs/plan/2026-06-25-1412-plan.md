> 历史记录（已完成，非现行事实源）：本计划记录 2026-06-25 的 HAL 结构化日志落地；当前实现、测试清单和验证准入以 [测试规范](../design/testing/testing-specification.md)、[HAL 契约](../design/contracts/hal-interface-protocol.md) 和 [日志契约](../design/contracts/log-interface-protocol.md) 为准。

目标：
把日志模块设计和 HAL 追踪链补齐，使 `requestId / durationMs / adapterCode / status / deviceId / resourceId / operation` 能从业务层调用贯通到 HAL 日志事件。

改动范围：
- 文档：`docs/design/contracts/hal-interface-protocol.md`
- 文档：`docs/design/contracts/log-interface-protocol.md`
- 公共 HAL 类型：`src/hal/include/hal/hal_types.h`
- HAL 服务接口：`src/hal/include/hal/i_hal_service.h`
- HAL 实现：`src/hal/src/hal_service.*`
- HAL 设备操作：`src/hal/src/hal_device.*`
- Adapter 桥接：`src/hal/src/c_abi_adapter.*` 后续接真实 DLL 时补齐
- Mock backend：保持可用，不强制它自己产生日志

实施步骤：
1. 在 `hal_types.h` 增加 `HalLogEvent`
   - 字段：`timestampUs, level, source, category, message, requestId, durationMs, status, adapterCode, deviceId, resourceId, operation, context`
   - 注册 `Q_DECLARE_METATYPE(hwtest::hal::HalLogEvent)`

2. 修改 `IHalService`
   - 保留旧 `logMessage(...)` 作为兼容信号
   - 新增：
     ```cpp
     void logProduced(const hwtest::hal::HalLogEvent& event);
     ```
   - 文档说明：新代码接 `logProduced`，旧 UI 可暂接 `logMessage`

3. 在 `HalService` 增加统一发日志函数
   - `emitLogEvent(const HalLogEvent&)`
   - 同时发 `logProduced(event)` 和旧 `logMessage(level, category, message, context)`
   - 旧 `context` 自动带上 `requestId / durationMs / status / adapterCode`

4. 在 `HalDevice` 增加操作日志包装
   - 用 `QElapsedTimer` 记录耗时
   - 调用 Adapter 后统一生成日志
   - 成功：`level=INFO, status=Ok`
   - 失败：`level=ERROR, status=toString(status.code), adapterCode=status.error.adapterCode`
   - 上下文带 `deviceId/resourceId/sessionId/physicalIndex/tags`

5. 覆盖 HAL 关键操作
   - `configureAd/readAd/writeDa`
   - `readDi/writeDo/waitEdge`
   - `openSerial/closeSerial/writeSerial/readSerial/transactSerial`
   - `openCan/closeCan/sendCan/receiveCan`
   - `openDevice/closeDevice/resetDevice/healthCheck`

6. 明确 Adapter 日志边界
   - 当前 `CAbiAdapter` 仍走 `MockAdapter`，不改 ABI
   - 文档补充：真实外部 Adapter 的 `host.log` 先进入 HAL，再转换为 `HalLogEvent(source="adapter")`
   - 若未来要让 Adapter 原生带 `requestId`，需升 `HAL_ADAPTER_ABI_VERSION`

7. 更新设计文档
   - `hal-interface-protocol.md`：把旧 `logMessage` 改成“兼容信号”，新增结构化日志事件
   - `log-interface-protocol.md`：补 HAL 到 LogService 边界图
   - 明确字段来源：
     - `requestId` 来自 `OperationOptions`
     - `durationMs` 由 HAL 计时
     - `adapterCode` 来自 `HalStatus.error.adapterCode`
     - `status` 来自 `HalStatusCode`

验证步骤：
1. 静态检查
   - `rg -n "HalLogEvent|logProduced|durationMs|adapterCode|requestId" src/hal docs/design`
   - 预期：接口、实现、文档均能搜到

2. 构建
   - 在项目构建目录运行 CMake 构建 HAL
   - 预期：`src/hal` 独立编译通过

3. 手动信号验证
   - 连接 `IHalService::logProduced`
   - 调用带 `OperationOptions{ .requestId = "REQ-001" }` 的 `openDevice/readAd/writeDa`
   - 预期日志含：
     - `requestId=REQ-001`
     - `durationMs >= 0`
     - `status=Ok` 或具体错误
     - 硬件失败时有 `adapterCode`

4. 回归验证
   - Mock backend 默认路径仍可初始化、开设备、读写 AD/DA/DI/DO、串口、CANFD
   - 旧 `logMessage` 订阅者仍能收到日志
