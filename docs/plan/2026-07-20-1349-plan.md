# Qt Widgets 双前端实现计划

> **Execution note:** 本计划可在当前工作区逐项执行，使用复选框跟踪进度。实施期间不修改 MB_DDF_v2 工程。
>
> **执行结果（2026-07-20）：** Task 1-8 的代码、测试、脚本和事实源同步已实施。Debug 与 Release 均完成全量构建及 136/136 CTest；TUI 以 `COM7` 会话覆盖完成 `load -> status -> quit`，GUI 以 offscreen 模式完成参数解析、启动和有序关闭，TUI/GUI 的 UDP Pass/Timeout/Stopped 等价性由自动测试覆盖。协议目录于 2026-07-20 15:05:38 +08:00 观测为 32 个 CSV。未执行可见窗口人工审阅、真实串口、真实网口或真实 DUT 验收；这些内容不得由自动模拟结果替代。

**目标：** 新增 Qt Widgets GUI，与现有 TUI 共享同一个应用控制器，具备等价的测试操作；通过 `hwtest.ps1 -Ui tui|gui` 选择前端。

**架构：** 保留独立的 `hwtest_tui.exe` 与 `hwtest_gui.exe`，分别使用 Console 子系统和 Windows GUI 子系统。二者只依赖 UI 无关的 `hwtest_app_core`，测试生命周期、配置、BIZ、算法、HAL 和日志继续由 `TestApplicationController` 统一组装。

**技术栈：** C++17、Qt 5.15 Widgets/Core/Network/SerialPort，Qt 6 fallback、CMake、PowerShell、GoogleTest/CTest。

## 一、设计约束

- `hwtest_tui`、`hwtest_gui`、`hwtest_pc_runner` 保持三个独立 EXE。
- GUI 不得 include 或 link HAL、BIZ、算法、Socket、`QSerialPort`。
- GUI 不调用 `waitForTerminal()`，通过 `snapshotChanged` 异步观察终态。
- GUI 启动后不自动 `prepare`、不打开硬件、不自动运行。
- `--control`、`--serial-port` 仍是会话覆盖，不写回 HAL JSON。
- UDP 接口继续保留；当前 MB_DDF_v2 实机交互仍只声明串口。
- 不引入 QML、Designer `.ui`、插件系统或新的状态管理框架。
- `hwtest.ps1` 无参数时继续显示帮助，不意外启动前端。

## 二、操作等价契约

| TUI 操作 | GUI 操作 | 统一控制器调用 |
| --- | --- | --- |
| `load` | “加载配置”按钮 | `loadConfigurations()`，再应用启动覆盖 |
| `controls` | 控制口下拉框 | `availableControls()` |
| `use <id>` | 选择控制口 | `selectControl()` |
| `ports` | 串口刷新按钮 | `availableSerialPorts()` |
| `port <name>` | 可编辑串口下拉框 | `selectSerialPort()` |
| `prepare` | “准备”按钮 | `prepare()` |
| `run` | “运行”按钮 | `start()` |
| `pause` | 暂停图标按钮 | `pause()` |
| `resume` | 继续图标按钮 | `resume()` |
| `stop` | 停止图标按钮 | `stop(5000)` |
| `status` | 常驻状态栏和进度条 | 最新 `ApplicationSnapshot` |
| `wait` | 异步等待，不阻塞窗口 | 订阅 `snapshotChanged` |
| `result` | 结果页 | `hasResult/verdict/rawData` |
| `disconnect` | “断开”按钮 | 有序 `stop -> shutdown` |
| `quit` | 关闭窗口 | 有序收尾后退出 |
| `help` | 自解释控件和工具提示 | 无业务动作 |

验收时，“等价”指相同输入配置和模拟目标产生相同的控制器阶段、最终判定、错误码和原始结果，不要求两种前端具有相同布局。
